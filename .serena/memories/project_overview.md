# プロジェクト概要

## プロジェクト名
**Flutter Hackathon Thema**

## プロジェクトの目的
モジュラー三層アーキテクチャで構築された、スケーラブルで保守性の高いFlutterアプリケーション。Flutterハッカソン用のテーマアプリケーションとして、明確な関心の分離と一方向の依存関係により、保守性とスケーラビリティを最大化する。

## 主な特徴
- ✅ 三層アーキテクチャ (App → Feature → Shared)
- ✅ モジュラー設計: 独立したFeatureモジュール
- ✅ 最新技術スタック: hooks_riverpod 3.x + go_router 16.x
- ✅ 開発インフラ整備: エージェント、コマンド、テンプレート完備
- ✅ FVM対応: Flutter バージョン管理
- ✅ 型安全: 厳格な静的解析（very_good_analysis）
- ✅ 包括的ドキュメント: アーキテクチャ、スタイルガイド、コントリビューションガイド

## プロジェクトのゴール
- スケーラブルな設計による機能追加の容易さ
- 保守性の高いコードベース
- 明確な関心の分離による開発効率の向上
- テスト可能な設計による品質保証

## エントリーポイント
- メインファイル: `lib/main.dart`
- アプリ起動コマンド: `fvm flutter run`

## システム環境
- OS: macOS (Darwin 24.5.0)
- Flutter Version Management: FVM (stable)
- Git リポジトリ: Yes
