# カスタムスラッシュコマンド

エージェントを簡単に呼び出すためのカスタムスラッシュコマンド一覧です。

## 利用可能なコマンド

### Flutter 開発

| コマンド | エージェント | 用途 |
|---------|------------|------|
| `/component` | flutter-component-specialist | UIコンポーネント開発 |
| `/state` | riverpod-state-expert | Riverpod状態管理 |
| `/router` | go-router-navigator | go_routerルーティング |

### Git / GitHub

| コマンド | エージェント | 用途 |
|---------|------------|------|
| `/commit` | github-commit-agent | コミット作成 |
| `/pr` | github-pr-creator | プルリクエスト作成 |
| `/issue` | github-issue-creator | イシュー作成 |

### ヘルプ

| コマンド | 用途 |
|---------|------|
| `/agents` | 全エージェント一覧表示 |

## 使い方

### 基本的な使い方

```
/component
新しいカスタムボタンコンポーネントを作成して
```

スラッシュコマンドを入力後、具体的なタスクを指示してください。

### コマンドなしでの起動

キーワードで自動起動することもできます：

```
"新しいボタンコンポーネントを作成" → flutter-component-specialist 起動
"状態管理を実装" → riverpod-state-expert 起動
"コミットして" → github-commit-agent 起動
```

## よくある使用例

### UIコンポーネント作成

```
/component
- 新しいカスタムカードコンポーネントを作成
- レスポンシブなボタンを実装
- 既存のウィジェットをリファクタリング
```

### 状態管理実装

```
/state
- ユーザー認証の状態管理を実装
- データローディング状態を管理
- フォーム状態を管理
```

### ルーティング設定

```
/router
- アプリのルーティングをセットアップ
- 認証ガードを実装
- タブナビゲーションを作成
```

### コミット作成

```
/commit
- この変更をコミット
- 機能実装をコミット
```

### PR作成

```
/pr
- この機能のプルリクエストを作成
- mainブランチへのPRを作成
```

### イシュー作成

```
/issue
- バグレポートを作成
- 機能リクエストを作成
```

## 開発フロー例

### 新機能開発の場合

```
1. /state
   状態管理を実装して

2. /component
   UIコンポーネントを作成して

3. /router
   ルーティングを設定して

4. /commit
   変更をコミットして

5. /pr
   プルリクエストを作成して
```

### バグ修正の場合

```
1. バグ修正を実装

2. /commit
   修正をコミットして

3. /issue (必要に応じて)
   バグレポートを作成して
```

## ヒント

### 具体的な指示が効果的

❌ "ボタンを作って"
✅ "新しいカスタムボタンコンポーネントを作成して"

❌ "状態管理して"
✅ "Riverpodでユーザー認証の状態管理を実装して"

### エージェント情報の確認

```
/agents
```

全エージェントの詳細情報が表示されます。

### 詳細ドキュメント

- エージェント詳細: `.claude/agents/README.md`
- 個別エージェント: `.claude/agents/*.md`

## トラブルシューティング

### コマンドが認識されない

1. `.claude/commands/` に該当ファイルが存在するか確認
2. Claude Codeを再起動してみる
3. `/agents` で利用可能なコマンド一覧を確認

### エージェントが起動しない

より具体的なキーワードを含めた指示を試してください：
- コンポーネント作成
- 状態管理実装
- ルーティング設定
- コミット作成

## カスタマイズ

新しいスラッシュコマンドを追加するには：

1. `.claude/commands/` に新しい `.md` ファイルを作成
2. frontmatter に `description` を追加
3. コマンド内容を記述

例：
```markdown
---
description: Custom command description
---

Command content here...
```
